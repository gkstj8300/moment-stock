## 2. 디자인 핵심 원칙

- **Zero-Lag Visual Feedback**: 모든 실시간 재고 변화는 100ms 이내에 시각적으로 반응해야 한다.
- **Semantic Clarity**: 색상이나 형태만으로 상태를 오해하지 않도록 의미론적 명확성을 최우선으로 한다.
- **Data Integrity First**: 초당 수천 건의 요청 속에서도 데이터의 정확성을 시각적으로 보장하는 디자인을 지향한다.

---

## 3. 시멘틱 컬러 시스템

### 3.1. 1:1 대응 및 의미론적 명명

- 컬러는 Hex값이 아닌 사용 목적과 UI 역할에 따라 네이밍하고 시스템화한다.
- 라이트 모드와 다크 모드의 컬러는 반드시 1:1로 대응되어야 한다.

### 3.2. 다크 모드 특화 규칙

- **배경색**: 눈의 피로를 줄이기 위해 순수 블랙(`#000`)보다 진한 쿨그레이(`#171B1C`) 사용을 권장한다.
- **레이어 계층**: 상단 레이어일수록 하단 배경보다 더 밝은 색을 적용하여 시각적 위계를 형성한다.
- **그림자**: 다크 모드에서는 그림자가 명확히 보이지 않으므로, 색상의 명도 차이(Elevated)만으로 레이어를 구분한다.

---

## 4. 컴포넌트 설계 전략

### 4.1. Flat & Compound 패턴 활용

- **Flat API**: 단순하고 반복적인 케이스(Badge, Button 등)에 사용하여 간결함을 유지한다.
- **Compound API**: 복잡하거나 확장이 잦은 케이스(ProductCard 등)에 적용하여 제품 팀이 직접 조합할 수 있는 유연성을 제공한다.

### 4.2. 아토믹 디자인

- **Atoms**: 개별 색상, 아이콘, 폰트 등 UI의 가장 작은 단위.
- **Molecules**: Atoms가 결합된 형태(입력 필드, 버튼 등).
- **Organisms**: Molecules의 집합으로 구성된 복잡한 섹션(내비게이션, 카드 리스트 등).

---

## 5. 실시간 UI 최적화 규칙

- **Optimistic UI & Rollback**: 구매 시 즉시 UI 상의 재고를 차감하되, 서버 검증 실패 시 자연스럽게 롤백되도록 설계한다.
- **Rendering Optimization**: 고주파 업데이트 상황에서 CPU 점유율을 낮추기 위해 부분 렌더링과 `React.memo`를 적극 활용한다.

---

## 6. UX Writing & 접근성

### 6.1. 잡초 뽑기 (Weed Cutting)

- 문장 내에서 기능을 하지 않는 불필요한 단어는 제거하여 간결한 보이스톤을 유지한다.
- 금융/커머스 전문 용어(Jargon)를 지양하고 누구나 이해하기 쉬운 용어로 대체한다.

### 6.2. 접근성 가이드

- **명도 대비**: 텍스트와 배경 간의 명도 대비는 최소 4.5:1(AA 등급) 이상을 유지한다.
- **색상 무관 인식**: 오류나 재고 상태를 색상만으로 표현하지 않고, 아이콘이나 텍스트를 반드시 함께 제공한다.

---

## 7. 레이아웃 및 간격

- Figma의 auto layout 값을 기반으로 Tailwind의 spacing 토큰(`--spacing-{n}`)만 사용한다.
- 임의의 arbitrary value(`[36px]` 등) 사용은 금지되며, 반드시 테마에 등록 후 사용한다.

---

## 8. 디자인 리소스 관리

- 디자인 토큰, 아이콘, CSS 스타일 등을 임의로 생성하지 않고 기존 리소스를 우선 탐색한다.
- 라이브러리 API나 컴포넌트 이름은 정확한 명칭을 확인한 뒤 사용한다.
